Observaciones
1 · Hosting — cPanel
  • 1 Crear subdominio cPanel → Dominios → Subdomains. Apuntar Document Root directo a la carpeta public/ del proyecto del cliente. Crítico
  • 2 Crear base de datos y usuario cPanel → MySQL Databases. Nombre sugerido: pventa_cliente. Asignar usuario con todos los privilegios. Crítico
  • 3 Subir archivos del proyecto Excluir: .env, node_modules/. El vendor/ puede subirse o regenerarse con composer install --no-dev. Importante
  • 4 Verificar permisos de carpetas Las carpetas storage/ y bootstrap/cache/ deben tener permisos 775. Sin esto Laravel no puede escribir logs ni sesiones. Crítico
  • 5 Activar SSL / HTTPS en el subdominio cPanel → SSL/TLS → Let's Encrypt (gratis). Obligatorio para que la cookie de sesión segura funcione. Crítico
2 · Archivo .env

Crear el archivo .env en la raíz del proyecto con los siguientes valores:

# Aplicación
APP_NAME="PVenta - Nombre Cliente"
APP_ENV=production
APP_KEY=<generar con: php artisan key:generate>
APP_DEBUG=false
APP_URL=https://cliente.tudominio.com

# Base de datos
DB_CONNECTION=mysql
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=pventa_cliente
DB_USERNAME=user_cliente
DB_PASSWORD=clave_segura

# Sesión
SESSION_LIFETIME=120
SESSION_SECURE_COOKIE=true
⚠ Importante: APP_DEBUG=false siempre en producción. Con true se exponen rutas, configuración y variables al navegador.
3 · Comandos artisan (SSH o terminal del hosting)
php artisan key:generate
php artisan migrate
php artisan db:seed
php artisan config:cache
php artisan route:cache
php artisan view:cache
Si el hosting no tiene SSH, ejecutar las migraciones desde phpMyAdmin importando el SQL generado, o usar un script PHP de un solo uso.
4 · Configuración inicial en BD (phpMyAdmin)

Después del seeder, ajustar los valores del cliente en la tabla globales:

-- Tipo de negocio del cliente
UPDATE globales SET valor_var = 'ALMACEN_PREVENTA' WHERE nom_var = 'TIPO_NEGOCIO';
-- Opciones: ALMACEN_PREVENTA | RESTAURANT

-- Verificar que el sistema está activo
UPDATE globales SET valor_var = '1' WHERE nom_var = 'SISTEMA_ACTIVO';

-- Suspender cliente (cuando no paga)
-- UPDATE globales SET valor_var = '0' WHERE nom_var = 'SISTEMA_ACTIVO';
5 · Verificación final
  • 1 Acceder al subdominio en el navegador Debe cargar la pantalla de login sin errores. Verificar
  • 2 Crear usuario administrador inicial Ingresar con el usuario del seeder y cambiar la contraseña por defecto. Verificar
  • 3 Probar generación de venta / preventa / comanda Confirmar que el flujo principal funciona en producción. Verificar
  • 4 Verificar subida de imágenes (logo y fotos de productos) Confirmar que los directorios public/img/logo_empresa y public/img/fotos_prod tienen permisos de escritura. Verificar
  • 5 Confirmar que el HTTPS funciona y el candado aparece en el navegador Obligatorio antes de entregar acceso al cliente. Verificar
  • 6 Registrar cliente en planilla de control Nombre, subdominio, BD, fecha vencimiento, estado: Activo. Verificar
